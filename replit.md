# ВафляМессенджер

Telegram-подобный мессенджер на русском языке.

## Стек
- **Backend**: Node.js + Express + Socket.io
- **База данных**: SQLite (better-sqlite3) — файл `data.sqlite`
- **Frontend**: Vanilla JS + CSS (без фреймворков)
- **Загрузка файлов**: Multer → `/uploads/`
- **Порт**: 5000
- **Workflow**: "Serve static" → `node script.js`

## Файлы
| Файл | Назначение |
|------|-----------|
| `script.js` | Express-сервер + Socket.io + все API-эндпоинты |
| `db.js` | SQLite-обёртка: все таблицы и функции |
| `client.js` | Весь клиентский JavaScript (SPA) |
| `index.html` | Разметка приложения |
| `styles.css` | Стили (CSS-переменные, компоненты) |

## Возможности

### Авторизация
- Вход/регистрация по номеру телефона + SMS-код
- Код регистрации по умолчанию: `935935`
- Выбор аватара (6 градиентов или загрузка фото)

### Личные чаты (DM)
- Отправка текста, файлов (фото, видео, аудио, документы)
- Реакции на сообщения (6 эмодзи)
- Удаление сообщений (у себя / у всех)
- Очистка переписки
- Блокировка пользователей
- Профиль собеседника (слайд-панель): медиа, ссылки, общие группы

### Группы
- Создание групп с фото, названием, описанием, участниками
- **Роли**: `owner` (создатель), `admin` (назначается владельцем), `member`
- Профиль группы (слайд-панель по клику на название):
  - Вкладка **Участники**: список с ролями, «...» меню у каждого
  - Вкладка **Медиа**: фото и видео из чата
  - Вкладка **Ссылки**: ссылки из сообщений
- **Добавление участников** (owner/admin)
- **Удаление участника** (owner/admin) → системное сообщение
- **Покинуть группу** (member/admin) → системное сообщение
- **Настройки группы** (owner/admin): фото, название, описание
- **Назначение администраторов** (только owner)
- Очистить сообщения для себя / для всех (owner/admin)
- Три точки `⋯` в шапке: только для личных чатов

### Системные сообщения
- Формат `__LINK__username|имя__ действие` → рендерится с кликабельным именем
- Открывает профиль участника при клике

### UI
- Цветовая схема: `--bg: #f0e4cc`, `--accent: #7c3aed`
- Фоновые изображения: `bg/auth.jpg`, `bg/chat.jpg`
- Адаптивная вёрстка (мобильный / десктоп)
- Единая кнопка «+» (dropdown: создать контакт / создать группу)

## API-эндпоинты (ключевые)

### Группы
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/groups` | Список групп пользователя |
| POST | `/api/groups/create` | Создать группу (multipart) |
| GET | `/api/groups/:id` | Инфо о группе + участники + моя роль |
| PUT | `/api/groups/:id` | Обновить группу (multipart, owner/admin) |
| GET | `/api/groups/:id/messages` | Сообщения + моя роль |
| POST | `/api/groups/:id/members/add` | Добавить участников |
| POST | `/api/groups/:id/members/remove` | Удалить участника |
| POST | `/api/groups/:id/leave` | Покинуть группу |
| POST | `/api/groups/:id/clear-me` | Очистить для себя |
| POST | `/api/groups/:id/clear-all` | Очистить для всех (owner/admin) |
| POST | `/api/groups/:id/set-role` | Изменить роль (только owner) |
| GET | `/api/groups/:id/media` | Медиа группы |
| GET | `/api/groups/:id/links` | Ссылки группы |

## Socket события
| Событие | Направление | Описание |
|---------|------------|----------|
| `dm:send` | клиент → сервер | Отправить личное сообщение |
| `dm:new` | сервер → клиент | Новое личное сообщение |
| `group:send` | клиент → сервер | Отправить в группу |
| `group:new` | сервер → клиент | Новое сообщение в группе (включая системные) |
| `group:joined` | сервер → клиент | Добавлен в группу |
| `group:removed` | сервер → клиент | Удалён из группы |
| `group:left` | сервер → клиент | Покинул группу |
| `group:cleared` | сервер → клиент | Сообщения очищены |
| `group:updated` | сервер → клиент | Группа обновлена |
| `group:role_changed` | сервер → клиент | Изменена роль участника |
| `msg:deleted` | сервер → клиент | Сообщение удалено |
| `reaction:update` | двустороннее | Реакция добавлена/убрана |
